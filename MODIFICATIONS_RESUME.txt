3. CHANGEMENTS APPORTÉS DÉTAILLÉS
─────────────────────────────────────────────────────────────────────────────
CONTRÔLEUR (SecurityController.php)

   Nouvelle route:
   ────────────────
   GET  /register   → Affiche le formulaire
   POST /register   → Traite la soumission
   
   Logique:
   1. Créer nouvelle entité Utilisateur
   2. Créer le formulaire avec RegistrationType
   3. Traiter la soumission (handleRequest)
   4. Si valide:
      - Hash le mot de passe
      - Persister en BDD (doctrine)
      - Afficher flash message
      - Rediriger vers login
   5. Si invalide:
      - Réafficher le formulaire avec erreurs
